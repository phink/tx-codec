# MichelinePack -- Verified Michelson PACK Codec

Formally verified encoder/decoder for the Michelson PACK binary format, targeting
Solidity (EVM) for [Tezos X](https://tezos.com/) cross-runtime calls.

## What this does

Solidity contracts on Tezos X need to encode EVM values into Michelson `PACK`
format (and decode the reverse) for cross-runtime atomic calls. This project:

1. **Specifies** the encoding in Lean 4 as pure functions over `List (Fin 256)`
2. **Proves** encode/decode roundtrip (bijection) for each type -- zero sorry, zero axioms
3. **Transpiles** a Solidity library (`MichelsonSpec.sol`) via a SolStmt IR transpiler
4. **Tests** the Solidity with 330 Foundry fuzz tests, 40 Halmos symbolic proofs, 96 asm-diff checks, and 414 octez-client compatibility tests

## Architecture

```
                   63 EQUIV THEOREMS                    SolStmt IR TRANSPILER
 Micheline.lean  =====================>  Implementation.lean  ==================>  MichelsonSpec.sol
 (pure spec,                             (offset-based,                            (reference Solidity,
  List (Fin 256),                         Except DecodeError,                       52/52 functions,
  roundtrip proofs,                       63 equiv theorems,                        zero templates)
  zero sorry)                             zero sorry)                                    |
                                                                                    Halmos + optimize
                                                                                         |
                                                                                         v
                                                                                  Michelson.sol
                                                                                  (production, asm)
```

**Trust boundary:** The Lean-to-Solidity step uses a real AST walker over
`SolExpr`/`SolStmt` IR (`Transpile.lean`), not template strings. All 52
functions are generated through this IR. The transpiler itself is not formally
verified; Halmos closes the gap by symbolically proving the compiled bytecode
satisfies the roundtrip property for all inputs.

### Lean files

| File | Purpose |
|------|---------|
| `MichelinePack/Micheline.lean` | **Core spec.** Pure functional encode/decode using `List (Fin 256)` for all 20 types. All roundtrip proofs (left and right) with zero sorry. Byte validity is structural -- enforced by `Fin 256`, no preconditions. |
| `MichelinePack/Implementation.lean` | **Offset-based Solidity-shaped encode/decode.** Uses `Except DecodeError` with 10 error types. 63 equivalence theorems to `Micheline.lean` (zero sorry). Functions map 1:1 to Solidity. |
| `MichelinePack/Transpile.lean` | **SolStmt IR transpiler.** 52/52 functions generated from IR (17 AST-transpiled + 35 IR-constructed). Zero template strings. Writes `MichelsonSpec.sol`. |
| `MichelinePack.lean` | Module root -- imports Micheline, Implementation, Transpile. |
| `Main.lean` | CLI: `lake exe michelinepack encode <n>`, `decode <hex>`, `emit`. |

## Quick start

```bash
# Build the Lean project
lake build

# Encode/decode from CLI
lake exe michelinepack encode 42      # -> zarith: 2a, packed: 05002a
lake exe michelinepack decode 05002a  # -> 42

# Emit the Solidity library
lake exe michelinepack emit > sol-test/src/MichelsonSpec.sol

# Run Solidity tests
cd sol-test && forge test
```

## Proven properties (Lean 4, zero sorry)

All byte lists use `List (Fin 256)` -- byte validity is structural, not a precondition.

### Zarith codec

| Property | Theorem | Precondition |
|----------|---------|--------------|
| `decode(encode(n)) = some(n, [])` | `Micheline.decode_encode` | -- |
| `encode(n) = bs` when `decode(bs) = some(n, [])` | `Micheline.encode_decode` | -- |
| `decodeInt(encodeInt(z)) = some(z, [])` | `Micheline.decodeInt_encodeInt` | -- |
| `encodeInt(z) = bs` when `decodeInt(bs) = some(z, [])` | `Micheline.encodeInt_decodeInt` | -- |

| `unpackInt(packInt(z)) = some z` | `Micheline.unpackInt_packInt` | -- |
| `packInt(z) = bs` when `unpackInt(bs) = some z` | `Micheline.packInt_unpackInt` | -- |
| `unpackBool(packBool(b)) = some b` | `Micheline.unpackBool_packBool` | -- |
| `packBool(b) = bs` when `unpackBool(bs) = some b` | `Micheline.packBool_unpackBool` | -- |

### Implementation equivalence

63 theorems in `Implementation.lean` prove that every offset-based function
computes the same result as the corresponding pure-spec function in
`Micheline.lean` (e.g., `Implementation.encode_eq`, `Implementation.packNat_eq`).

### Halmos (40 symbolic proofs)

| Category | Count | Description |
|----------|-------|-------------|
| Asm equivalence | 18 | `asm(x) == spec(x)` for all inputs (nat, int, bool, mutez, timestamp, pair, or, option, list) |
| Spec roundtrip | 12 | `unpack(pack(x)) == x` for all inputs (nat, int, bool, string, mutez, timestamp, pair, or, option, list) |
| Binary roundtrip | 6 | `unpack(pack(x)) == x` for address, bytes, key_hash, key, signature, chain_id |
| Adversarial | 4 | Arbitrary bytes: asm rejects iff spec rejects (nat, int) |
